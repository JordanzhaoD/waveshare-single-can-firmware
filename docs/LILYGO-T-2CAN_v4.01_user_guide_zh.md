# LILYGO T-2CAN Tesla CAN Dashboard 使用指南

适用版本：v4.01  
适用设备：LILYGO T-2CAN / ESP32-S3 双 CAN 固件  
适用对象：普通车主、测试人员、安装人员

## 1. 这套固件能做什么

这套固件把 LILYGO T-2CAN 变成一个车载 CAN 控制与监控面板。你可以通过手机或车机浏览器打开控制面板，查看设备状态、切换激活模式、调整驾驶风格、配置网络和 DNS 黑名单、上传 OTA 固件，并使用 CAN2 相关测试工具。

主要功能包括：

- 查看 CAN1 / CAN2 状态、帧率、运行时间、设备版本。
- 控制 FSD 相关模块总开关。
- 选择 Auto / Legacy / HW3.0 / HW4.0 激活模式。
- 选择驾驶模式：Auto、Sloth、Chill、Normal、Hurry、MAX。
- 设置速度偏移策略。
- 配置热点、上游 WiFi、DNS 黑名单和白名单。
- 上传 OTA 固件。
- 查看 CAN 帧、录制 CAN 数据、调试 CAN 控制器。
- 使用 CAN2控制 页面进行灯光、拨杆、Service Mode 等实验功能测试。

## 2. 使用前注意事项

这不是普通娱乐设备，而是连接车辆 CAN 总线的调试设备。请务必遵守以下原则：

- 先在停车状态下测试，不要在行驶中首次操作新功能。
- 不确定功能含义时，不要随意打开实验开关。
- 涉及 FSD 注入、灯光连续爆闪、MAX/V14、AP/EAP 免打扰等功能，都需要实车验证。
- 刷写固件或修改网络配置前，确认设备供电稳定。
- 如果出现异常，优先关闭“模块总开关”，必要时断开设备供电。

界面中带有“未验证 / 实验”的功能，表示目前更适合测试人员验证，不建议普通用户日常启用。

## 3. 第一次连接设备

1. 给 LILYGO T-2CAN 上电。
2. 用手机或车机连接设备 WiFi 热点。
3. 默认热点通常为：

```text
SSID: CHANGE_ME_SSID  (在 platformio_profile.h 里修改)
密码: CHANGE_ME_PASS  (在 platformio_profile.h 里修改)
```

4. 打开浏览器，访问：

```text
http://100.100.1.1/
```

5. 如果打不开页面，请检查：

- 手机是否已经连接到设备热点。
- 浏览器是否自动跳到了移动网络。
- 设备是否刚重启，还在启动中。
- 是否曾修改过 AP SSID 或密码。

## 4. 手机端和车机端界面

### 车机端 / 桌面端

车机端左侧是完整导航栏，包含：

- 模块配置
- 激活模式
- 驾驶模式
- 速度偏移
- CAN2控制
- FSD防御
- OTA升级
- 网络设置
- CAN工具

顶部状态栏会显示连接状态、FSD 状态、实验功能提示、运行时间和白天 / 夜间主题切换。

### 手机端

手机端底部是主要导航：

- 状态
- 模式
- 速度
- 网络
- 更多

“更多”里面放置 OTA升级、CAN2控制、FSD防御、CAN工具等低频或危险操作。

手机端顶部有“白天 / 夜间”切换按钮。白天模式适合户外强光，夜间模式适合车内暗光环境。

## 5. 模块配置

“模块配置”是首页，也是日常最常用的页面。

你可以在这里查看：

- 设备是否在线。
- CAN 是否正常。
- 当前激活模式。
- 当前驾驶模式。
- 是否有异常告警。
- 当前固件版本。

### FSD 总开关

这是手机首页最重要的开关，用于控制 FSD 相关功能是否启用。

建议使用方式：

1. 先确认 CAN1 状态正常。
2. 再打开 FSD 总开关。
3. 如果出现异常，先关闭这个开关。

### 核心激活程序

这里包含两个独立开关：

- FSD-V14 模式
- 模块总开关

两者是不同功能，不应互相联动关闭。普通使用时优先只操作“模块总开关”。FSD-V14 / MAX 类功能属于实验能力，需要实车确认。

### 系统状态

系统状态会显示：

- CAN1 状态
- CAN2 状态
- RX / TX 帧数
- 帧率
- TX 错误
- 设备运行时间

如果 CAN1 或 CAN2 显示离线，不一定代表设备坏了，可能是：

- 车辆没有上电。
- 接线未连接到对应 CAN 总线。
- 总线当前没有数据。
- CAN2 外置控制器未检测到有效帧。

## 6. 激活模式

左侧菜单“激活模式”用于选择车辆硬件平台适配方式。

可选项：

- Auto：自动判断，普通用户优先选择。
- Legacy：旧版平台或兼容模式。
- HW3.0：Tesla HW3 平台。
- HW4.0：Tesla HW4 平台。

推荐：

- 不确定车辆硬件版本时，选择 Auto。
- 已明确是 HW3 或 HW4 时，再手动指定。
- 修改后观察系统状态和 CAN 状态是否稳定。

页面里的“AP 状态恢复”用于控制重启后是否恢复上次 AP 相关配置。普通用户建议保持默认。

## 7. 驾驶模式

“驾驶模式”用于选择速度和行为策略的整体风格。

可选项：

- Auto：自动策略。
- Sloth：更保守。
- Chill：温和。
- Normal：普通。
- Hurry：更积极。
- MAX：最大 / V14 实验策略。

建议：

- 日常测试优先使用 Auto、Chill 或 Normal。
- Hurry 和 MAX 更适合短时验证，不建议长期默认开启。
- MAX / V14 属于实验功能，需要人工实车验证。

### FSD 注入控制

这里可以控制 FSD 注入相关开关。

如果界面要求二次确认，请认真确认后再执行。普通用户不要在行驶过程中反复切换。

### 开机自动启用

开启后，设备重启后会尝试恢复上次的启用状态。

建议：

- 测试阶段关闭。
- 确认稳定后再考虑开启。

### 紧急控制

包含重启设备、重置计数等操作。

重启设备会短暂断开控制面板连接，属于正常现象。执行前请确认车辆处于安全状态。

## 8. 速度偏移

“速度偏移”用于管理速度相关策略。

可选策略：

- fixed：固定策略。
- auto：自动策略。
- custom：自定义策略。

普通用户建议使用 auto。custom 适合知道自己在调什么的测试人员。

### HW3 自定义速度

这是需要实车验证的功能。你可以设置不同实际速度对应的目标速度。

使用建议：

- 不要一次改太多。
- 每次只调整一个小范围。
- 记录修改前后的表现。
- 如果出现异常，恢复默认或关闭自定义速度。

### 高速映射

高速映射会影响更高速度区间的行为。普通用户不建议修改。

### 速度编码方式

用于选择 CAN 总线速度编码方式。除非明确知道车辆需要哪种编码，否则保持默认。

## 9. CAN2控制

“CAN2控制”主要面向测试和验证。它包含 CAN2 状态、ID 列表、灯光特技、拨杆测试和 Service Mode。

### CAN2 ID 列表

这里显示 CAN2 总线上发现的 ID。它能帮助判断 CAN2 是否接线正常。

如果列表为空，可能原因：

- CAN2 没有连接。
- 车辆当前没有对应数据。
- 接线位置不对。
- MCP2515 控制器未收到有效帧。

### 灯光特技

包含爆闪开关、次数、频率、后雾灯策略等配置。

重要提醒：

- 灯光连续爆闪属于实验功能。
- 不要在公共道路上随意测试。
- 测试前确认周围安全，避免影响其他车辆和行人。

### 拨杆测试

用于模拟部分方向盘拨杆操作。执行前界面会要求确认。

普通用户不要频繁使用。测试人员应记录执行时间、CAN2 状态、车辆响应。

### Service Mode

Service Mode 是服务或调试相关功能，普通用户不建议开启。

## 10. FSD防御

“FSD防御”用于配置一组防护或兼容策略。

包含：

- 防御总开关。
- slew rate 限制。
- 仿生方向盘。
- 声音警告抑制。
- 速度免打扰。
- AP/EAP 兼容。

推荐：

- 普通用户只开启基础防御。
- 仿生方向盘、速度免打扰、AP/EAP 兼容属于实验或未验证功能。
- 修改后观察异常告警和 CAN 状态。

### slew rate 限制

它用于避免速度偏移变化过快。一般建议开启。

### 声音警告抑制

这类功能可能影响提醒体验，请谨慎使用。

## 11. OTA升级

“OTA升级”用于通过浏览器上传新固件。

使用步骤：

1. 打开 OTA升级 页面。
2. 查看当前固件版本。
3. 点击选择固件文件。
4. 选择 OTA app 固件，例如：

```text
LILYGO-T-2CAN_v4.01_ota_app_xxxxx.bin
```

5. 点击开始上传。
6. 等待上传完成和设备重启。
7. 重新连接热点并刷新页面。

注意：

- OTA 只能上传 OTA app 固件，不要上传 full merged 全量固件。
- full merged 固件用于 USB 串口全量刷机。
- 上传中不要断电。
- 如果 OTA 后页面打不开，等待 30 秒再重新连接热点。

## 12. 网络设置

“网络设置”把热点、上游 WiFi、STA-AP 网关、DNS 黑白名单放在一起。

### WiFi 热点

这里可以修改设备热点：

- AP SSID
- AP 密码
- 是否隐藏热点

保存后通常需要重启设备。修改前建议记下新密码，避免忘记后无法连接。

### WiFi 联网

用于让设备连接外部 WiFi。

使用步骤：

1. 点击扫描网络。
2. 选择目标 WiFi。
3. 输入密码。
4. 点击保存或连接。
5. 查看状态是否变为已连接。

### 外部中转 WiFi 测试

用于测试设备能否连接指定外部 WiFi。测试失败时，请检查密码、信号强度和路由器设置。

### STA-AP 网关

开启后，设备可以在热点和上游 WiFi 之间转发网络流量。

普通用户建议：

- 不需要中转网络时关闭。
- 如果网页变慢，可以打开网络性能模式。

### DNS 黑名单 / 白名单

这里用于管理 DNS 过滤规则。

页面提供两种策略：

- 保守模式：规则更多，过滤范围更完整。
- 激进模式：规则更少，只保留最小过滤集。

注意这里的“激进”不是过滤更多，而是策略更冒险、更少拦截。  
如果希望尽量拦截相关域名，选择保守模式。

你也可以手动编辑：

- 黑名单：需要拦截的域名，每行一个。
- 白名单：始终允许的域名，每行一个。

修改后点击“保存规则”。

### DNS 测试

输入域名后点击测试，可以查看该域名是否会被当前规则拦截。

## 13. CAN工具

“CAN工具”面向调试人员。

### CAN 引脚配置

显示当前固件使用的 CAN 引脚，例如：

- CAN1 TX / RX
- CAN2 CS
- SPI SCK / MISO / MOSI
- MCP RST

普通用户只查看即可，不建议随意修改。

### CAN Sniffer

用于实时查看 CAN 帧。

常用操作：

- 输入 ID 或关键词过滤。
- 暂停刷新，方便查看。
- 观察 ID、方向、数据和时间。

### CAN Recorder

用于录制 CAN 数据并下载 CSV。

使用步骤：

1. 设置录制帧数限制。
2. 点击开始录制。
3. 等待录制完成或手动停止。
4. 点击下载 CSV。

### CAN Controller

显示控制器错误状态，例如 RX Errors、TX Errors、EFLG。

如果错误持续增加，可能是：

- 波特率不匹配。
- 接线错误。
- CAN H / CAN L 接反。
- 总线终端电阻或供电异常。

### Debug 日志

开启后会增加调试信息。普通用户平时关闭即可。

## 14. 全量刷机和 OTA 固件怎么区分

发布资产通常有两类：

### OTA app 固件

文件名一般包含：

```text
ota_app
```

用途：

- 在控制面板 OTA升级 页面上传。
- 不需要 USB 串口。
- 适合设备已经能正常打开网页时使用。

### full merged 全量固件

文件名一般包含：

```text
full_merged
```

用途：

- 用 USB 串口刷入。
- 包含 bootloader、partition、OTA data 和 app。
- 适合首次烧录、系统损坏恢复、彻底刷新。

不要把 full merged 文件上传到 OTA 页面。

## 15. 常见问题

### 打不开控制面板

检查：

- 手机是否连接设备热点。
- 地址是否输入为 `http://100.100.1.1/`。
- 设备是否刚重启。
- 热点名称或密码是否被修改。

### 页面显示未连接

可能是 CAN 未接入，或车辆当前没有上电。先确认设备供电，再确认 CAN 接线。

### CAN1 正常但 CAN2 没有数据

可能是 CAN2 接线位置不对，或者该总线当前没有有效帧。可以进入 CAN2控制 或 CAN工具查看 ID 列表。

### 修改网络后连不上热点

可能是 AP SSID 或密码修改后忘记。需要通过串口重新刷机或恢复配置。

### OTA 后页面还是旧版本

尝试：

- 强制刷新浏览器。
- 清理浏览器缓存。
- 重新连接热点。
- 查看 OTA升级 页面里的固件版本。

### 点按钮没有反应

可能原因：

- 设备网络断开。
- 当前页面状态没有刷新。
- 后端功能需要 CAN 在线。
- 某些实验功能只保存配置，不一定立即有车辆响应。

## 16. 推荐日常使用流程

普通用户建议按以下顺序使用：

1. 连接设备热点。
2. 打开控制面板。
3. 在“模块配置”确认 CAN1 / CAN2 状态。
4. 在“激活模式”选择 Auto。
5. 在“驾驶模式”选择 Auto、Chill 或 Normal。
6. 在“网络设置”按需要配置 DNS 规则。
7. 不需要时关闭实验功能。
8. 升级固件时优先使用 OTA app 文件。

## 17. 测试人员建议记录的信息

每次实车验证建议记录：

- 固件版本。
- 车辆年份和平台：Legacy / HW3 / HW4。
- 使用的接口和针脚。
- CAN1 / CAN2 是否在线。
- 当前激活模式。
- 当前驾驶模式。
- 是否开启 FSD防御。
- 是否开启实验功能。
- 问题发生时间和复现步骤。
- CAN Recorder 导出的 CSV。

这些信息能帮助后续定位问题。

## 18. 安全恢复建议

如果不确定当前状态是否安全：

1. 关闭模块总开关。
2. 关闭 FSD 总开关。
3. 关闭实验功能。
4. 重启设备。
5. 必要时断开设备供电。
6. 使用 full merged 固件通过 USB 重新全量刷机。

优先保证车辆和人员安全，再进行功能排查。

## Legacy Real Wiring 实车预检

在实车测试 Legacy 通道功能前，通过 Dashboard 确认以下状态：

### 前置检查

1. **Handler 确认**：`/status` 中 `effectiveHw` 必须为 `0`（显示 `LEGACY`）。
2. **FSD 策略确认**：`fsdDiag.policy` 必须显示 `legacy_stable`。
3. **FSD 注入**：
   - `fsdDiag.mux0.tx` 应在 0x3EE mux0 注入时递增。
   - `fsdDiag.mux0.lastSkip` 应为 `sent`。
   - `fsdDiag.mux1.tx` 在 Stable 模式下必须保持 `0`，`lastSkip` 为 `disabled_in_stable`。
4. **Service 维修模式**：
   - `serviceDiag.tx.txOk` 应为 `4`（4 帧突发），`txErr` 为 `0`。
   - `lastData` 应匹配 enter（`80`）/ exit（`00`）命令。
   - `vehicleResponse` 为 `unknown`，直到实车观察到车辆反应。
5. **Wheel DND**：
   - `wheelDndDiag.native3c2Seen` 必须为 `true` 后才能发送。
   - 未收到原生 0x3C2 时，`baseFrameMode` 为 `waiting_native`（安全默认）。

### Stable vs Experimental

- **默认 Stable**：mux0 仅写 bit46，mux1 不发送，速度 profile 不写入。这是最稳定的 FSD 注入模式。
- **Experimental**：通过 `/legacy_fsd_config` 开启。可启用 mux1 注入、速度 profile 写入、视觉限速清除。仅用于 A/B 对比测试。

### Dashboard UI 卡片

- **Legacy Runtime Matrix**：显示当前 HW 模式、AP Gate、OTA Guard 等运行时状态。
- **Legacy FSD Stable**：FSD 注入策略和 mux0/mux1 诊断数据。
- **Service 0x339**：Service 维修模式 TX 证据。
- **Wheel DND 0x3C2**：原生帧检测和发送序列状态。
- **Legacy Support Matrix**：标注 Legacy 不支持的功能（Bionic Steering、Ban Shield 等）。
